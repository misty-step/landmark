Ticket 017: Fleet Rollout Safety

Make fleet adoption decide what each repo is, what mode it should use, what secrets are actually needed, and what evidence must exist before rollout continues.

Assumptions

  • Remote PR creation must stay guarded and auditable; no implicit mutation from a planning command.
  • Classification should be explicit in scan and plan artifacts so operators can override bad inferences.
  • Secret readiness should depend on mode. Local and generic CI adoption should not be blocked by GitHub Action secrets.

Evidence

  • fleet_adoption_planner replay proves classification, integration modes, secret minimization, receipts, and dashboards.
  • Schema and README contracts name new fields and refusal rules.
  • bin/gate, adversarial review, hosted parity, and post-merge checks close the ticket.
