Backlog 014 / Agent-Native Contracts

Make Landmark self-describing enough for a cold agent to configure, run, verify, and debug without scraping prose.

Chosen shape: keep the product boundary in the Rust CLI. Add a checked schema registry, a machine-readable describe --json command, stable JSON failure envelopes, and a replay scenario that proves an agent can consume those contracts.

Goal And Assumptions

Goal: expose stable runtime contracts for manifests, synthesis status, replay result, fleet plans, release-note entries, and run evidence.

Assumption: 014 is a contract and verification ticket, not a redesign of release synthesis quality or provider retries.

Acceptance

Versioned schemas exist, describe --json names commands/providers/schemas/examples, agent commands have JSON output or explicit preview policy, and failures emit stable machine fields.

Verify

replay-action --scenario agent_native_contracts, check-action-contract, schema parse checks, and full bin/gate.

Communicate

Report schema files, CLI surface, replay evidence, critic findings, PR checks, squash merge, and post-merge run status.

Stop If

Schema drift cannot be checked locally, runtime output needs a breaking public contract, or a mutating command cannot honestly preview and has no documented reason.

Current State

Typed Rust structs exist for most artifacts, but no checked schemas/*.schema.json files exist and agents must infer stdout/stderr contracts from README examples.

Change Shape

Add one CLI command and one schema registry. Keep GitHub-specific behavior behind adapters. Avoid new language seams and avoid shell-generated schemas.

Proof Surface

The replay scenario invokes the current executable, reads the checked schemas, validates describe --json, exercises a JSON error, and runs a local provider release.

Alternatives

  • Use a Rust schema-generation crate. Faster long term, but adds dependency and derive churn today.
  • Publish static schemas only. Simpler, but repeats the existing stale-prose failure mode.
  • Make every command fully JSON first. Cleaner eventually, too broad for one backlog item.

Risk Controls

  • Contract checker parses schema JSON and fails when README/schema/runtime tokens drift.
  • Replay validates the cold-agent integration guide from the current executable.
  • Failure envelope redacts token-like content and includes code, stage, retryability, action, and context.
