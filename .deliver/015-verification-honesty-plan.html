Backlog 015 / Verification Honesty

Make Landfall’s reliability and security claims executable instead of aspirational.

Chosen shape: harden the Rust HTTP chokepoint, prove action subcommand coverage with a meta-check, and expand replay around side effects that the GitHub Action depends on.

Stop Conditions

Stop if token-safe HTTP cannot be tested without leaking secrets, if a side-effect path needs real production credentials, or if a README claim cannot be mapped to a local oracle.

Acceptance

Bounded retrying external calls, no secret-bearing curl argv, redacted failures, replay coverage for side-effecting subcommands, and docs claims tied to checks.

Proof

replay-action --scenario http_resilience_policy --scenario action_side_effect_coverage, focused unit tests, full bin/gate, hosted CI.

Non-goals

No production GitHub/Slack/webhook calls, no new runtime language, no broad action shell rewrite unless required by the oracle.

Execution Plan

1 harden

Refactor curl_json into a bounded helper that feeds URL, auth headers, and body through curl config on stdin and retries transient errors.

Evidence: unit test proves args contain no configured secret and config does.

2 cover

Add replay scenarios for HTTP 429/5xx/timeout behavior and a meta-test mapping action dist/landfall subcommands to replay coverage.

Evidence: replay packet names the covered commands.

3 docs

Update README reliability/security text to name the exact checks instead of vague guarantees.

Evidence: contract check fails when claims drift.

4 ship

Run full gate, adversarial review, refresh Linux binary through hosted CI if needed, squash merge, and monitor master.

Evidence: local + hosted receipts.
