Deliver 013: Pipeline-Neutral Landmark

Make GitHub one provider path by adding a local, tokenless release run that uses the same Rust runtime and replay evidence model as the GitHub Action.

Status: execution plan Owner: Codex Updated: 2026-06-15 Premise: backlog.d/013

Recommendation

Add a provider-neutral run command with --provider local as the first concrete provider. It should read git tags/CHANGELOG/manifest, compute a release evidence packet, write markdown/plaintext/HTML/JSON/RSS artifacts, and avoid GitHub tokens entirely.

Definition Of Done

  • landmark run --provider local produces release evidence and artifacts from a disposable git repo.
  • Replay has a canonical local-provider scenario.
  • README and backlog explain the non-GitHub path.
  • PR is reviewed, green, squash-merged, and master is monitored.

Stop Conditions

  • Need to replace semantic-release entirely in this ticket.
  • Need GitHub or external LLM credentials to prove local mode.
  • Replay cannot exercise the new command locally.

Visual Map

Current State

Backfill can derive notes from git ranges and write artifacts, but normal release orchestration is centered on action.yml and GitHub release mutation.

Change Shape

Promote the git-range source and artifact writing into run --provider local; keep GitHub full mode intact.

Proof Surface

cargo test --locked, landmark replay-action --scenario local_provider_run, bin/gate, fresh reviewer, PR checks, post-merge runs.

Scope

  • Goal: tokenless local release pipeline with evidence.
  • Non-goal: complete GitLab/Forgejo adapters in this ticket.
  • Invariant: existing GitHub Action behavior and outputs survive.
