Ticket 019: Attach To Existing Release Workflows

Make fleet open-prs update the release workflow that already owns versioning or publishing, instead of rendering a second competing Landmark release workflow.

Boundary

Keep the change in the Rust fleet scan, plan, and dry-run PR renderer. No Node, Python, or new shell orchestration.

Design

Carry scanned release-workflow metadata into the fleet plan, render explicit workflow patches for release-please and changesets, and block existing semantic-release workflows for operator choice.

Oracle

Replay fixtures prove no duplicate release jobs or standalone landmark-release.yml for existing release-please and changesets workflows.
