Ticket 020: Backfill-First Fleet Adoption

Make active application and library repositories with packages, no release tags, and no release automation eligible for a safe manifest/local-artifacts adoption PR before any release-mutating workflow is proposed.

Boundary

Keep the change in the Rust fleet planner, dry-run PR renderer, fleet schema, README, and replay fixtures. Do not add Node, Python, or a new release orchestration layer.

Design

Promote backfill-first from blocked to ready only for no-release application/library repos with package topology. The PR writes .landmark.yml and receipts, never a GitHub release workflow.

Proof

fleet_adoption_planner replay must cover Rust, TypeScript, Go, Python, and multi-package no-release fixtures and prove skipped non-release repos remain skipped.

Quality System

  • Standards: Rust CLI remains the product core; the action wrapper stays out of this slice.
  • Proof methods: docs first, failing tests, code, cargo fmt, focused tests, bin/replay-action --scenario fleet_adoption_planner, bin/gate, hosted CI.
  • Critic topology: after local green, give a fresh critic only the diff plus ticket oracle and resolve blockers before PR closeout.
  • Stop rules: stop if the lane would add release mutation for no-release repos, lower gates, or require GitHub secrets for local artifact adoption.
